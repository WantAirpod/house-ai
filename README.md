# home-decision-ai

부부가 실거주 아파트 매수를 준비하기 위한 부동산 인텔리전스 플랫폼입니다.

목표는 Railway에 배포 가능한 웹/API 서비스, Postgres 기반 데이터 저장소, 일간/주간 리포트 생성 파이프라인, Notion 게시 연동을 갖춘 실거주 매수 의사결정 시스템을 만드는 것입니다.

## 사용자 목표

- 1년 이내 실거주 아파트 매수
- 최대 예산 10억 ~ 10.5억
- 네이버 본사(정자동) 출퇴근 고려
- 관심 지역: 용인 수지, 용인 기흥, 용인 처인, 서울 내 판교 출퇴근 가능 지역
- 관심 평형: 59, 84
- 선호 연식: 12년 이하, 입지가 좋으면 구축도 검토

## 제품 범위

포함:

- 사용자/배우자 선호도 YAML 설정
- 관심 지역 및 관심 단지 YAML 관리
- Railway 배포 가능한 FastAPI 앱
- Railway Postgres 연동 경계
- 일간/주간/알림 리포트 프롬프트 템플릿
- 스케줄러 워커 구조
- Notion 게시 연동 경계
- 수동 입력 데이터와 자동 수집 데이터를 모두 수용하는 구조

부분 구현 또는 향후 보강 영역:

- 호가 수집기: 네이버페이 부동산 관측값을 1차 호가 출처로 사용하고, 수동 관측 CSV 또는 별도 검증 소스 입력 필요
- 커뮤니티/블로그/뉴스 반응 수집기
- K-apt 공동주택 기본/상세 정보와 랭킹 파이프라인 연결
- 실제 보행 경로 기반 역 도보시간 계산
- 프로덕션 인증/권한

## 설치

Python 3.12와 uv가 필요합니다.

```bash
cd home-decision-ai
uv sync
```

## 로컬 실행

설정 확인 CLI:

```bash
uv run home-decision-ai
```

웹/API 서버:

```bash
uv run uvicorn home_decision_ai.api.app:create_app --factory --reload
```

헬스 체크:

```bash
curl http://localhost:8000/health
```

## Railway 배포

Railway 프로젝트에 Postgres를 추가하고, 앱 서비스에 다음 환경변수를 설정합니다.

- `DATABASE_URL`
- `HOME_DECISION_AI_ENV=production`
- `HOME_DECISION_AI_CONFIG_DIR=config`
- `OPENAI_API_KEY` 또는 사용하는 LLM provider 키
- `NOTION_API_KEY`
- `NOTION_PARENT_PAGE_ID`

이 저장소는 `Procfile`을 통해 웹 프로세스를 실행합니다.

## 데이터 운영 방식

- `data/manual/`: 사람이 직접 정리한 매물, 호가, 커뮤니티 반응, 정책 메모
- `data/raw/`: 향후 API 또는 파일 기반 원천 데이터 저장 위치
- `data/processed/`: 정규화된 분석용 데이터 저장 위치
- `reports/daily/`: 일간 리포트
- `reports/weekly/`: 주간 리포트
- `reports/alerts/`: 조건 기반 알림 리포트

## 데이터 원칙

- 출처와 관측일이 없는 가격 데이터는 분석에 사용하지 않습니다.
- 실거래가는 국토교통부 실거래가 공개시스템 같은 공식 출처를 우선합니다.
- 호가와 매물 수는 네이버페이 부동산을 우선 확인하고 관측일, 출처 URL, 중복 가능성 메모를 함께 저장합니다.
- 커뮤니티 반응은 단독 근거가 아니라 보조 리스크 신호로만 사용합니다.
- AI 의견은 `verified` 또는 `manually_checked` 데이터에만 강한 결론을 붙입니다.

## TOP10 후보 리서치 기준

최종 TOP10 후보는 실거래가만으로 확정하지 않습니다. 각 후보 단지는 아래 신호를 함께 추적합니다.

- 실거래가: 최근 거래가, 기간 내 저점/고점 위치, 거래 빈도
- 호가: 네이버페이 부동산 기준 현재 최저호가, 실거래 대비 호가갭, 매물 수, 관측일
- 역 접근성: K-apt 지하철역명/지하철역 거리, 카카오 추정 도보시간, 20분 초과 여부
- 단지 규모: K-apt 세대수/단지분류, 200세대 이하 여부, 500세대 이상 주상복합 여부, 아파트/오피스텔형 여부
- 생활 품질: 실거주 후기, 주차, 소음, 동간거리, 관리상태, 커뮤니티 시설
- 지역 여론: 개발 호재, 공급 리스크, 학군/상권/교통 변화

## 단지 규모 및 주거유형 필터

실거주 매수 후보에서는 나홀로 아파트와 소규모 단지를 보수적으로 제외합니다.

- 200세대 이하 단지는 원칙적으로 제외합니다.
- 세대수 미확인 단지는 `세대수 검수 미완료`로 표시하고 최종 확정 후보로 보지 않습니다.
- 오피스텔, 도시형생활주택, 오피스텔형 아파트, 아파텔 성격 후보는 제외합니다.
- 500세대 이상 주상복합은 아파트성 단지로 보고 후보에 포함할 수 있습니다.
- 201~499세대 아파트는 역세권, 브랜드, 관리상태, 주차, 커뮤니티 수준을 추가 검토한 뒤 보류/유지 여부를 판단합니다.
- TOP10 후보는 세대수와 주거유형 출처를 반드시 기록합니다. 이 값이 없으면 순위가 높아도 검수대기 후보입니다.

## K-apt 공공데이터 활용

공동주택 기본 정보제공 서비스와 공동주택 상세 정보제공 서비스를 단지 검수의 공식 기준으로 사용합니다.

- 기본 정보: 단지명, 법정동주소, 도로명주소, 세대수, 동수, 단지분류, 시공사, 시행사, 사용승인일, 전용면적별 세대현황
- 상세 정보: 지하철호선, 지하철역명, 지하철역 거리, 주차대수, 승강기대수, CCTV대수, 부대복리시설, 편의시설, 교육시설
- 세대수/단지분류/사용승인일은 TOP10 확정 전 필수 검수 항목입니다.
- K-apt 지하철역 거리는 공공데이터 기준으로 우선 사용하고, 실제 보행 시간은 지도 길찾기로 재확인합니다.

샘플 조회:

```bash
PYTHONPATH=src python3 scripts/fetch_kapt_info.py --bjd-code 4146510100 --name-contains 성복
PYTHONPATH=src python3 scripts/fetch_kapt_info.py --kapt-code KAPT_CODE
```

공공데이터포털 키는 서비스별 활용신청 상태가 다를 수 있습니다. 국토부 실거래 API 키가 동작하더라도 K-apt 기본/상세 정보 서비스가 별도 승인되지 않았으면 응답 오류가 날 수 있으므로, `공동주택 기본 정보제공 서비스`, `공동주택 상세 정보제공 서비스`, `공동주택 단지 목록제공 서비스` 활용신청 상태를 확인합니다.

최종 TOP10 후보에 대해서는 단지별로 블로그, 뉴스, 커뮤니티 글, 부동산 분석 글 등 외부 자료를 최소 10개 이상 확인하고 추적합니다.

- 각 자료는 제목, URL, 작성일 또는 확인일, 출처 유형을 기록합니다.
- 홍보성 글과 실제 거주 후기성 글을 구분합니다.
- 같은 내용을 복사한 중복 글은 1개 근거로만 계산합니다.
- 10개 미만이면 해당 단지는 `외부자료 부족`으로 표시하고 확정 판단을 보류합니다.
- 외부자료는 결론을 대체하지 않고 실거주 리스크를 보강하는 근거로 사용합니다.

## 역세권 판단 기준

역 접근성은 매수 선호도에 직접 반영합니다.

- 도보 10분 이하: 강한 선호
- 도보 15분 이하: 선호
- 도보 20분 이하: 약한 감점
- 도보 20분 초과: 강한 감점

현재 도보시간은 카카오 로컬 API의 단지 좌표와 가장 가까운 지하철역 사이 직선거리를 기준으로 추정합니다. 실제 보행 경로와 차이가 날 수 있으므로 최종 판단 전 지도 길찾기로 재확인합니다.
